# Wazuh 기반 실시간 엔트로피 계산을 통한 SSH 브루트포스 공격 패턴 분석

---

## 초록 (Abstract)

본 연구는 오픈소스 SIEM 플랫폼인 Wazuh와 Slack을 연동한 실시간 보안 알림 시스템을 구축하고, Shannon Entropy와 Approximate Entropy(ApEn)를 활용하여 SSH 브루트포스 공격의 패턴 변화를 분석한다. 기존 룰 기반 탐지 방식은 사전에 정의된 임계값에 의존하기 때문에 공격 속도를 조절하는 우회 시도에 취약하다. 본 연구에서는 슬라이딩 윈도우(크기 100) 기반의 엔트로피 계산을 통해 정상 트래픽 구간과 공격 구간의 수치 변화를 실시간으로 기록하고, 두 구간의 엔트로피 값을 비교 분석하였다. 실험 결과, 공격 시작 시점을 기준으로 Shannon Entropy와 ApEn 모두 뚜렷한 변화를 보였으며, 이는 엔트로피가 공격 패턴 분석의 유효한 지표가 될 수 있음을 시사한다.

---

## 1. 서론

### 1.1 연구 배경

네트워크 보안 위협이 고도화됨에 따라 단순 룰 기반 탐지 시스템의 한계가 부각되고 있다. SSH 브루트포스 공격은 가장 빈번하게 발생하는 공격 유형 중 하나로, 기존 시스템은 단위 시간당 로그인 실패 횟수가 임계값을 초과할 때만 알림을 발생시킨다. 그러나 공격자가 시도 속도를 늦추거나 다수의 IP를 분산하여 사용하는 경우 탐지가 어렵다.

정보 이론의 Shannon Entropy는 데이터의 불확실성과 다양성을 수치화하는 지표로, 네트워크 이상 탐지 분야에서 활용 가능성이 주목받고 있다. 공격 트래픽은 정상 트래픽에 비해 패턴이 단조롭고 반복적이기 때문에 엔트로피 값이 낮아지는 특성을 보인다.

### 1.2 연구 목적

본 연구의 목적은 다음과 같다.

1. Wazuh SIEM과 Slack을 연동한 실시간 보안 알림 시스템 구축
2. 슬라이딩 윈도우 기반 Shannon Entropy 및 ApEn 실시간 계산 구현
3. 정상 트래픽과 SSH 브루트포스 공격 구간의 엔트로피 변화 비교 분석

## 2. 이론적 배경

### 2.1 정보량과 Shannon Entropy

Claude Shannon은 어떤 사건이 발생했을 때 얻는 정보의 양을 해당 사건의 발생 확률로 정의하였다. 확률이 낮은 사건일수록 발생 시 더 많은 정보를 제공한다.

특정 사건 x의 정보량 I(x)는 다음과 같이 정의된다.

```
I(x) = -log₂ P(x)
```

Shannon Entropy H는 확률 분포 전체의 평균 정보량으로, 분포가 균등할수록 높고 특정 값에 집중될수록 낮아진다.

```
H(X) = -Σ P(x) log₂ P(x)
```

### 2.2 Approximate Entropy (ApEn)

Approximate Entropy는 시계열 데이터의 규칙성을 측정하는 지표다. Shannon Entropy가 값의 분포를 보는 반면, ApEn은 값의 순서와 패턴의 반복성을 측정한다. 값이 낮을수록 수열이 규칙적으로 반복됨을 의미한다.

### 2.3 보안 이상 탐지에서의 엔트로피 활용

공격 트래픽은 동일한 출발지 IP, 동일한 목적지 포트, 반복적인 실패 패턴을 보이기 때문에 엔트로피 값이 낮아진다. 반면 정상 트래픽은 다양한 사용자와 서비스에서 발생하므로 엔트로피가 상대적으로 높다. 이 차이를 이용하면 임계값 없이도 공격 패턴을 수치로 관찰할 수 있다.

## 3. 시스템 구현

### 3.1 실험 환경

| 구성 요소 | 사양 |
|-----------|------|
| 호스트 OS | Rocky Linux 9 (Docker) |
| SIEM | Wazuh 4.9.0 (wazuh-manager, wazuh-dashboard, wazuh-indexer) |
| 에이전트 | Wazuh Agent 4.9.0 (호스트 설치) |
| 공격 VM | Kali Linux (IP: 192.168.190.128) |
| 알림 채널 | Slack (ZenSOC 워크스페이스) |

### 3.2 구현 단계

**Phase 1 — Slack Webhook 연동**
Wazuh가 생성하는 `alerts.json` 파일을 실시간으로 감시하여 Rule Level 7 이상의 alert를 Slack으로 전송하는 Python 스크립트를 구현하였다.

**Phase 2 — 실제 공격 탐지 확인**
Kali Linux에서 Hydra를 이용한 SSH 브루트포스 공격을 수행하고 Wazuh 탐지 및 Slack 알림 수신을 확인하였다.

**Phase 3 — 중복 알림 제거**
동일한 `(rule_id, agent_name)` 조합이 60초 윈도우 내에 반복 발생할 경우 카운트만 증가시키고, 윈도우 종료 시 발생 횟수를 포함한 요약 알림 1건만 전송하도록 구현하였다. 실험 결과 32건의 alert가 3건의 요약 알림으로 압축되었다.

**Phase 4 — 엔트로피 계산**
매 alert 발생 시 최근 100개의 alert를 슬라이딩 윈도우로 유지하며 다음 항목의 엔트로피를 계산하고 JSONL 형식으로 저장한다.

| 항목 | 측정 대상 |
|------|-----------|
| shannon_rule_id | 발생 Rule ID의 분포 다양성 |
| shannon_level | Alert Level의 분포 다양성 |
| shannon_srcip | 출발지 IP의 분포 다양성 |
| apen_level | Alert Level 수열의 규칙성 |

## 4. 실험 및 결과

### 4.1 데이터 수집

| 구간 | Alert 수 | 설명 |
|------|----------|------|
| 정상 구간 | 0 ~ 293 | 시스템 정상 운영 중 발생한 로그 |
| 공격 구간 | 293 ~ 4069 | Hydra SSH 브루트포스 공격 중 발생한 로그 |
| 총계 | 4069 | |

### 4.2 결과 분석

공격 시작 시점(index 293)을 기준으로 모든 엔트로피 지표에서 뚜렷한 변화가 관찰되었다.

**Shannon Entropy — Source IP**
정상 구간에서 0에 가까운 값을 유지하다가 공격 시작 직후 1.0 수준으로 급격히 상승하였다. 이는 단일 출발지 IP에서 대량의 트래픽이 집중 발생하는 브루트포스 공격의 특성을 반영한다.

**Shannon Entropy — Level**
정상 구간에서 0을 유지하다가 공격 구간에서 0.8~1.0으로 안정화되었다. 동일한 레벨의 alert가 반복 발생하는 공격 패턴이 수치로 나타난 결과다.

**Approximate Entropy — Level**
정상 구간에서 0을 유지하다가 공격 직후 0.6까지 상승한 후 0.3~0.5 수준으로 안정화되었다. 공격 초기에 다양한 패턴이 혼재하다가 점차 반복적인 패턴으로 수렴하는 과정이 관찰된다.

### 4.3 고찰

실험 결과, 엔트로피 기반 분석은 임계값 설정 없이도 공격 시작 시점과 공격 패턴의 변화를 수치로 포착할 수 있음을 확인하였다. 특히 Shannon Entropy — Source IP 지표는 공격 시작과 동시에 즉각적인 변화를 보여 조기 탐지 지표로서의 가능성을 시사한다.

다만 본 연구는 단일 공격자(단일 IP)의 SSH 브루트포스 공격만을 대상으로 하였으며, 분산 공격(Distributed Brute Force)이나 다른 공격 유형에 대한 추가 실험이 필요하다.

## 5. 결론

본 연구에서는 Wazuh SIEM 기반의 실시간 엔트로피 계산 시스템을 구현하고, 정상 트래픽과 SSH 브루트포스 공격 구간의 Shannon Entropy 및 ApEn 변화를 비교 분석하였다. 실험 결과 공격 시작 시점을 기준으로 모든 엔트로피 지표에서 유의미한 변화가 관찰되었으며, 이는 엔트로피가 룰 기반 탐지를 보완하는 공격 패턴 분석 지표로 활용될 수 있음을 보여준다.

향후 연구에서는 엔트로피 임계값 기반의 자동 탐지 로직 구현, 다양한 공격 유형에 대한 실험, 그리고 머신러닝과의 결합을 통한 탐지 정확도 향상을 목표로 한다.
